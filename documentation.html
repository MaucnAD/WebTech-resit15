Documentation of my api

Requests

The web API is based on REST principles. Web API uses appropriate HTTP verbs for each action:

Method Action
GET Retrieves resources
POST Creates resources
PUT Changes and/or replaces resources
DELETE Deletes resources

There are 2 types of GET, GET (a specific item) or you can GET the full data set.

Our products have the following attributes in the database (in ascending order):

  1. author
  2. alt
  3. tags
  4. description
  5. image

These attributes mean the following:

  • author: the name of the notable person. (string)
  • alt: a description of the image. (string)
  • tags tags that you might associate with the author. (string)
  • description: a description of why the author was notable. (string)
  • image: the url of the image of the product (string)

GET (individual product) Description
Product ID Numerical description of the product
URL http://localhost:3000/notable/:id
Here you can input the name of an author to search for them in the table.
Possible response codes
  • 200: OK - The request has succeeded. The client can read the result of the request in the body and the headers of the response.
  • 202: Accepted - The request has been accepted for processing, but the processing has not been completed.
  • 204: No Content - The request has succeeded but returns no message body.
  • 404: Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
  • 500: Internal Server Error.

Responses

Web API returns all response data as a JSON object.

Response status codes (summary)

Status code Description
200 OK - The request has succeeded. The client can read the result of the request in the body and the headers of the response.
201 Created - The request has been fulfilled and resulted in a new resource being created.
202 Accepted - The request has been accepted for processing, but the processing has not been completed.
204 No Content - The request has succeeded but returns no message body.
304 Not Modified.
400 Bad Request - The request could not be understood by the server due to malformed syntax.
403 Forbidden - The server understood the request, but is refusing to fulfill it.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
500 Internal Server Error.
